KONTEKST I CILJ SOFTVERSKOG PROJEKTA

	KONTEKST
	Projekat se razvija kao interaktivna collaborative desktop (ili 		web) aplikacija koja omogućava korisnicima da u realnom 		vremenu crtaju, dodaju objekte i komuniciraju putem chat-a.
	Aplikacija je namenjena za onlajn sastanke, predavanja ili timski 		rad, gde više korisnika istovremeno želi da radi na istoj 		tabli.
	Sistem funkcioniše u distribuiranom okruženju, sa klijentima koji 		se povezuju na centralni server (Blackboard) ili koriste 		message broker za real-time sinhronizaciju.
	Obezbeđuje kontrolu pristupa, tako da organizator sastanka određuje 		ko može da crta i menja sadržaj table.

	Cilj softverskog projekta
	Omogućiti više korisnika da simultano i u realnom vremenu crtaju i 		rade na zajedničkoj tabli.
	Implementirati real-time chat između korisnika, kako bi mogli da 		komuniciraju tokom crtanja.
	Obezbediti perzistenciju promena i undo/redo funkcionalnost, kako 		bi se sačuvala istorija rada i izbegao gubitak podataka.
	Primena arhitekturnih obrazaca (MVVM/MVC, Blackboard, Event push) 		radi modularnosti, skalabilnosti i lakog održavanja.
	Omogućiti jednostavno proširenje sistema u budućnosti (novi objekti 		za crtanje, dodatne funkcionalnosti).

ARHITEKTURNO-SPECIFIČNI ZAHTEVI

Distribucija i komunikacija
	Sistem mora biti distribuiran u client-server arhitekturi.
	Komunikacija između klijenata i servera mora koristiti message-			passing ili message broker (ZeroMQ / RabbitMQ).
	Server / Blackboard mora omogućiti push event propagation ka svim 		pretplaćenim klijentima u realnom vremenu.

	Obrasci za organizaciju i modularnost
		Blackboard obrazac za centralizovano aktivno skladište 				stanja table i istorije crteža.
		MVVM obrazac (ili MVC) za organizaciju GUI-ja i logike:
			Model čuva stanje table i chat poruka.
			ViewModel/Controller prima korisničke događaje i 				ažurira Model.
			View prikazuje stanje i automatski se osvežava.

	Implicit Invocation / Event-based push:
		Klijenti ne proveravaju stanje table (pull), već tabla 				(Blackboard) odlučuje kada šalje update.
		Eventi se razlikuju po tipu: DrawEvent, ChatEvent, 			RoleChangeEvent.

	Upravljanje pristupom i rolama
		Sistem mora omogućiti definisanje role-ova:
			Organizator meet-a ima pravo da crta i dodeljuje 				prava ostalim korisnicima.
			Ostali korisnici mogu dobiti ili izgubiti pravo 				crtanja u realnom vremenu.

		Event-based mehanizam mora propagirati promenu u pravima 			pristupa svim relevantnim klijentima.

	Skalabilnost i performanse
		Sistem mora biti sposoban da podrži 10–50 istovremenih 				korisnika.
		Event push mora osigurati da promene budu propagirane do 			svih korisnika za najviše 2 sekunde.

	Perzistencija i repository
		Svi događaji (crteži, chat poruke) moraju se perzistirati u 			repository radi undo/redo i istorije.
		Event processing ne sme blokirati GUI niti uzrokovati 				gubitak događaja.
		Slanje podataka preko mreže mora biti delta-based, tj. samo 			nove promene se šalju.

	Pouzdanost i dostupnost
		Sistem mora pouzdano obezbediti integritet podataka: 				nijedna promena ne sme biti izgubljena.
		Server i broker moraju biti robusni i otporni na greške, 			dok god server radi, sistem je dostupan.

	Modifikabilnost i proširivost
		Arhitektura mora omogućiti dodavanje novih tipova objekata 			za crtanje bez promene postojećeg koda.
		Event-based i modularna struktura olakšava dodavanje novih 			tipova događaja (npr. buduća podrška za audio ili 			nove alatke).

	Ukratko:
	Arhitekturno-specifični zahtevi u tvom projektu se svode na:
		Client-server distribuciju
		Blackboard + Event push za centralizovano aktivno skladište 			i real-time update
		MVVM / MVC za GUI modularnost
		Role-based pristup i kontrolu prava
		Perzistenciju događaja i pouzdanu komunikaciju
		Skalabilnost, performanse i modifikabilnost

IZABRANI APLIKACIONI OKKVIRI I BIBLIOTEKE
1. Klijentska komponenta (desktop aplikacija)
	Jezik: C#
	Framework: WPF (Windows Presentation Foundation)
		Razlog izbora:
			Moderni GUI sa podrškom za Canvas, toolbar i 					stilizovane kontrole.
			Lako implementira MVVM obrazac (Model-View-					ViewModel), što omogućava jasnu separaciju 				GUI-ja i logike.
			Dobro podržava data binding, što olakšava 					automatsko osvežavanje View-a pri promenama 				u Model-u.
	Obrasci koje primenjujemo u klijentu:
		MVVM:
			Model: čuva stanje table, chat poruka i korisničkih 				prava.
			ViewModel: prima događaje sa View-a i prosleđuje ih 				Model-u, obrađuje dolazne event-e od 					servera.
			View: Canvas za crtanje, chat prozor i toolbar, 				koji se automatski osvežava prilikom 					promena u Model-u.

		Event-based push: klijent se pretplaćuje na događaje sa 			Blackboard servera ili message broker-a i reaguje 			kada stignu novi događaji.
	Biblioteke i alati:
		Reactive Extensions (Rx.NET) – za elegantno upravljanje 			eventima i asinhronim tokovima podataka.
		Newtonsoft.Json – za serijalizaciju objekata (crteža i 				poruka) pri slanju preko mreže.
		ZeroMQ ili RabbitMQ .NET klijent – za komunikaciju sa 				serverom / brokerom.

2. Serverska komponenta (distribuirani server / Blackboard)
	Jezik: C# (može biti konzolna ili servis aplikacija)
	Framework: .NET Console
	Obrasci koje primenjujemo:
		Blackboard / Repository: centralizovano aktivno skladište 			stanja table, istorije crteža i chat poruka.
		Implicit Invocation / Event push: server (Blackboard) 				odlučuje kada će klijentima poslati nove događaje.
	Biblioteke i alati:
		ZeroMQ / RabbitMQ .NET klijent – za distribuciju događaja 			prema svim klijentima.
		Entity Framework Core – za perzistenciju podataka u 				lokalnoj ili udaljenoj bazi podataka.

3. Dodatne napomene
MVVM obrazac se primenjuje samo na klijentsku GUI logiku, dok server 	koristi Blackboard + Event push arhitekturu.
Ovakva kombinacija omogućava:
	Jasnu modularnost i separaciju odgovornosti
	Jednostavno proširenje (dodavanje novih tipova objekata ili 			događaja)
	Pouzdanu i real-time distribuciju događaja svim korisnicima
